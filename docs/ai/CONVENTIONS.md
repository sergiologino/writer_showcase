# Конвенции проекта

## Память и контракт

- Единственный источник истины о проекте для ассистентов — файлы в `docs/ai/`. История чатов не является памятью.
- В память не попадают: логи выполнения, stacktrace, черновые гипотезы и «поток мыслей» — только итоговые факты: что сделано, почему, что дальше.

## Изменения кода

- Любое значимое изменение кодовой базы сопровождать актуализацией `CURRENT_STATE.md`.
- Архитектурные повороты — запись в `DECISIONS.md`; при отмене решения — пометка **Superseded** и приведение кода в соответствие.
- Завершение задачи — краткая строка в `CHANGELOG_AI.md`.
- Не ломать существующий рабочий функционал без явного запроса; без массовых рефакторингов «ради чистоты».

## Тесты и сборка

- На фичи писать автотесты; они выполняются при каждой сборке (`apps/web`: `vitest run` внутри `npm run build`; `apps/backend`: `mvnw verify`; с репозитория: **`npm run build`**). Падение теста — разбор причины, исправление, повтор сборки.

## Расширение памяти

- При росте доменов — отдельные файлы в `docs/ai/domains/` (по домену или по сервису), без дублирования очевидного из overview.

## Стиль кода

- Следовать принятому в репозитории стилю (после появления кода): форматирование, именование, слой API vs domain — уточнять по существующим модулям.
